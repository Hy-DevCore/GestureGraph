# 知识图谱手势交互展示系统 (GestureGraph) 

## 1. 产品愿景
打造一个沉浸式的知识图谱探索工具。用户无需鼠标键盘，通过摄像头捕捉手部动作，以“隔空取物”的方式在庞大的知识网络中进行漫游和检索。该项目致力于成为 Hongyi Dev 旗下的明星开源项目，展示极客级别的前后端结合与 AI 交互能力。

## 2. 核心交互链路
1.  **唤醒与校准**：用户打开页面，授权摄像头。系统识别到手部特征点并进行空间坐标初始化。
2.  **全向漫游模式 (Omnidirectional Panning)**：
    * **动作**：五指张开 (Open Palm)。
    * **反馈**：捕捉手掌中心的移动轨迹。系统支持 360 度全向漫游，手掌的上下、左右以及**任意角度的斜向移动**，都将实时且丝滑地映射为 3D 画布相机的 X 和 Y 轴的联合偏移。这种体验类似于在空中操控一个无阻尼的摇杆。
    * **阻尼与惯性**：为了防止手部微小抖动导致画面闪烁，引入阻尼系数（Smoothing / Lerp），让视角的移动具有平滑的物理惯性。
3.  **锁定与选择 (Select & Click)**：
    * **动作**：从张开到握拳 (Closed Fist)。
    * **反馈**：冻结画布移动，以当前屏幕中心（或手部虚拟指针悬停处）的节点为目标，触发“Click”事件，弹出侧边栏展示节点的详细内容（关联关系、属性、文本解析）。

## 3. 技术栈选型
采用前后端分离架构，确保渲染性能与后端数据查询的解耦：

* **前端 (交互与可视化)**：
    * 核心 HTML/CSS/Vanilla JS（或轻量级框架）。
    * **手势识别**：`MediaPipe Hands` (Google 开源，纯客户端运行，极低延迟)。
    * **图谱渲染**：`3d-force-graph` (WebGL)。在处理大量节点和物理碰撞时性能优于传统 2D D3.js。
* **后端 (API 与数据)**：
    * **Web 框架**：`FastAPI` (Python) - 异步高性能，自带 Swagger UI。
    * **图数据库**：`Neo4j` - 存储实体和关系，提供原生的 Cypher 图谱查询能力。
* **部署架构**：
    * 前端使用 Nginx 作为 Web 服务器与反向代理。
    * 后端与数据库通过 Docker-compose 进行多阶段构建与容器化部署。
