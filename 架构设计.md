## 2. 核心技术点说明 (Vibe Coding 提示词背景)

### 2.1 手势识别与全向映射逻辑 (`gesture.js`)
前端的核心是将 MediaPipe 的 `HandLandmarker` 结果转化为带方向和大小的连续向量。

* **状态判定**：实时计算 5 个指尖到手腕的相对距离。全部张开判定为 `OPEN_PALM`，全部握紧判定为 `CLOSED_FIST`。
* **全向坐标映射 (核心难点)**：获取手掌中心点 (Landmark 9) 在屏幕坐标系下的 `(x, y)` 坐标。
    * **计算向量**：当处于 `OPEN_PALM` 状态时，不要对轴进行锁定。直接计算当前帧与上一帧的坐标差值，得到一个二维向量 `(dx, dy)`。这天然支持了对角线等任意角度的斜向滑动。
    * **平滑处理 (Lerp)**：MediaPipe 捕获的数据存在高频抖动。必须在 `gesture.js` 中引入线性插值 (Linear Interpolation)。不要将原生 `(dx, dy)` 直接赋给相机，而是让相机的目标位置逐步逼近手势的当前推移量：`camera.x = camera.x + (targetX - camera.x) * easingFactor` (建议 `easingFactor` 为 0.1 左右)。
    * **视角更新**：将计算好的平滑偏移量持续注入给 `3d-force-graph` 的相机平移 API，实现全向摇杆般的拖拽响应。
* **锁定与选择**：当状态从 `OPEN_PALM` 迅速切换为 `CLOSED_FIST` 时，清空移动向量冻结画布，并向后端 FastAPI 触发选中节点的具体详情查询。

### 2.2 图谱数据获取 (`app.js` & `routes.py`)
* **后端 API (`GET /api/graph`)**：
    * FastAPI 连接 Neo4j，执行核心 Cypher 查询：`MATCH (n)-[r]->(m) RETURN n, r, m LIMIT 200`（可配合手势坐标加入空间检索逻辑）。
    * 将结果格式化为 `3d-force-graph` 识别的标准 JSON：`{ "nodes": [...], "links": [...] }`。
* **前端渲染**：通过 `fetch` 获取到 JSON 后，传入 `ForceGraph3D().graphData(data)` 进行渲染。

### 2.3 部署容器化方案 (Docker)
1.  **Neo4j 容器**：使用官方镜像，配置持久化数据卷 `volumes` 存储图谱数据。
2.  **后端 FastAPI 容器**：基于 `python:3.11-slim`，安装依赖，使用 `uvicorn` 启动并暴露 `8000` 端口。
3.  **前端 Nginx 容器**：将 `frontend` 静态文件映射或拷贝至 Nginx HTML 目录，配置 `nginx.conf` 将 `/api/` 路径反向代理转发至 FastAPI 容器，完美规避前后端分离带来的 CORS 问题。

## 3. 开发起步建议
1.  将此文档交给 Trae，要求其优先生成 `frontend/index.html` 和 `frontend/js/gesture.js`。
2.  要求 Trae 实现纯前端的 Demo：开启摄像头，控制台能根据手部开合准确输出状态，并基于张开的手势输出平滑处理后的 `dx, dy` 向量。
3.  前端逻辑跑通后，再要求生成后端 FastAPI 和 Neo4j 的连接代码。
